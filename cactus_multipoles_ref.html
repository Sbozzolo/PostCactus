Reference on postcactus.cactus_multipoles¶

This module provides access to data saved by the multipoles thorn.

The main class is MultipolesDir, which is typically accessed through SimDir instances.

class postcactus.cactus_multipoles.MultipoleAllDets(data, l_min=0)¶

This class collects available surfaces with multipole data.

It works as a dictionary in terms of spherical surface radius, returning a MultipoleOneDet object. Iteration is supported, sorted by ascending radius. You can iterate over all the radii and all the available l and m with a nested loop.

Not intended for direct use.

Variables

  • radii (float) – Available surface radii.

  • r_outer (float) – Radius of the outermost detector

  • l_min (int) – l smaller than l_min are dropped

  • outermost (MultipoleOneDet) – Outermost detector

  • available_lm (list of tuples) – Available components as tuple (l,m).

  • available_l (list) – List of available “l”.

  • available_m (list) – List of available “m”.

Data is a list of tuples with structure (multipole_l, multipole_m, extraction_radius, [timeseries])

has_detector(mult_l, mult_m, dist)¶

Check if a given multipole component extracted at a given distance is available.

  • dist (float) – Distance of detector

class postcactus.cactus_multipoles.MultipoleOneDet(dist, data, l_min=0)¶

This class collects multipole components of a specific variable a given spherical surface.

Multipoles are tightly connected with gravitational waves, so morally a sphere where multipoles are computed is a “detector”. Hence, the name of the class.

It works as a dictionary in terms of the component as a tuple (l,m), returning a TimeSeries object. Alternatively, it can be called as a function(l,m). Iteration is supported and yields tuples (l, m, data), which can be used to loop through all the multipoles available.

The reason we allow for l_min is to remove those files that are not necessary when considering gravitational waves (l<2).

Not intended for direct use.

Variables

  • dist (float) – Radius of the sphere

  • radius (float) – Radius of the sphere

  • l_min (int) – l smaller than l_min are dropped

  • available_l (set) – Available l values

  • available_m – Available m values

  • available_m – set

  • available_lm – Available (l, m) values

  • available_lm – set of tuples

  • missing_lm – Missing (l, m) values to have all from l_min to l_max

  • missing_lm – set of tuples

Data is a list of tuples with (l, m, timeseries).

l smaller than l_min are dropped

total_function_on_available_lm(function, *args, l_max=None, **kwargs)¶

Evaluate function on each multipole and accumulate the result.

total_function_on_available_lm will call function with the following arguments:

function(timeseries, mult_l, mult_m, dist, *args, **kwargs)

If function does not need some paramters, it should use take the *args argument to ignore the additional paramters that are always passed (l, m, r).

Values of l larger than l_max are ignored.

function can take additional paramters passed directly from total_function_on_available_lm (e.g. pcut for FFI).

Params function

Function that has to be applied on each monopole

Returns

Sum of function applied to each monopole

Return type

return type of function

class postcactus.cactus_multipoles.MultipolesDir(sd)¶

This class provides acces to various types of multipole data in a given simulation directory.

This class is like a dictionary, you can access its values using the brackets operator. The output is a MultipoleAllDets, which has a full multipolar description for all the available radii. Files are lazily loaded. If both h5 and ASCII are present, h5 are preferred. There’s no attempt to combine the two.

Alternatively, you can access variables with get() or with fields.var_name.
